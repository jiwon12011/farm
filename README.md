## 2. 세계관 & 스토리

### 시놉시스
주인공은 번아웃에 시달리던 도시 직장인. 어느 날, 5년 전 돌아가신 **할머니의 편지**가 뒤늦게 도착한다 — "반딧불 마을의 농장을 너에게 맡긴다. 부엌 서랍의 레시피북을 찾으렴."

도착한 **반딧불 마을**은 이상하다. 한때 '맛의 마을'로 불렸지만 지금은 반딧불이 사라지고, 작물은 시들하고, 주민 대부분이 떠났다. 그리고 할머니의 레시피북은 **대부분의 페이지가 은은히 빛나는 덩굴 문양으로 잠겨 있다.**

특정 요리를 완성할 때마다 페이지가 한 장씩 풀리며 **할머니의 기억 조각**이 재생된다. 기억을 따라가다 보면 진실이 드러난다 — 마을의 풍요는 숲의 정령 **루미**와 할머니의 오랜 약속이었고, 할머니가 떠난 뒤 약속이 끊기며 정령들(반딧불)이 잠들어 마을이 빛을 잃은 것. 주인공은 레시피를 복원하고 마을을 수리해 정령들을 하나씩 깨우고, 마지막 페이지의 요리 — **'할머니의 달빛 스튜'** — 를 루미에게 대접하며 약속을 다시 잇는다.

## 3. 코어 시스템 설계

### 3.1 농사
- 밭 1칸 = 타일 1개. 시작 6칸 → 최대 24칸(확장 구매).
- 상태 머신: `빈 밭 → 씨앗 → 새싹 → 성장 → 성숙(수확 가능) → (시듦 없음: 힐링 게임이므로 수확물이 썩지 않고 대기)`
- **물주기(선택적 가속)**: 물을 주면 해당 성장 단계 속도 **1.5배**. 안 줘도 자람(방치 보장). 비 오는 날은 전 밭 자동 급수.
- **품질 ★1~3**: 물주기 횟수·비료·정령 축복으로 확률 상승. 판매가 ×1.0/×1.4/×2.0, 요리에 품질 전이.
- **다회 수확 작물**(토마토, 포도 등): 첫 수확 후 짧은 재생 시간으로 반복 수확.
- **비료**: 퇴비통에 잡초·작물 찌꺼기 투입 → 2시간 후 비료. 사용 시 ★확률 +25%.

### 3.2 목장
- 동물 구매(초원) → 축사 배치 → **실시간으로 생산물 축적**(상한 3개, 방치 핵심).
- **쓰다듬기**(하루 1회) + **먹이 주기** → 행복도 → 생산 속도·품질 상승. 먹이는 밀·옥수수 등 자가 재배 작물 (농사↔목장 순환).
- 새끼 → 성체 성장(실시간). 새끼는 저렴하지만 생산까지 대기 필요.

### 3.3 요리 (게임의 심장)
- 조리 도구 4종: **프라이팬**(1장) → **냄비**(3장) → **오븐**(3장) → **절임통**(4장) 순서로 해금.
- 실시간 조리(1분~2시간). 도구당 1개씩 동시 조리, 도구 업그레이드로 대기열 +2.
- **레시피 해금 3경로**: ① 스토리(레시피북 페이지) ② 보리 아주머니 힌트 구매 ③ **자유 실험** — 재료 2~3개 조합 투입, 정답이면 신규 레시피 발견(도감 등록 + 발견 보너스), 오답이면 '수상한 죽'(정령은 이것도 좋아함 — 실패가 벌이 아닌 재미).
- 요리 등급: 일반/고급/희귀/전설. **판매가 = 재료 원가 합 × 2.0~2.5** — 요리가 항상 원물 판매보다 이득이도록 고정 원칙.

### 3.4 판매 (3단계 수익 구조)
1. **노점(즉시)**: 아무거나 즉시 판매. 기본가. 초반용.
2. **주문 게시판(액티브 보너스)**: 주민·행인의 의뢰 3~5건 상시. 지정 요리/작물 납품 시 **기본가 ×1.5 + 호감도**. 4시간마다 갱신.
3. **식당 '반디네 부엌'(방치 수익 코어)**: 진열대 슬롯(4→12칸)에 요리를 채워두면 **오프라인 동안 손님이 자동 구매**. 판매 속도는 마을 복구도·요리 등급에 비례. "자기 전에 진열대 채우기"가 핵심 습관 루프.

### 3.5 마을 복구 (타이쿤 메타 진행)
돈 + 지정 재료를 망치에게 전달 → 실시간 공사(1~24시간) → 완공 시 **마을 배경이 눈에 띄게 밝아지고 반딧불 수 증가**(시각적 진행도).

| 순서 | 시설 | 해금 효과 |
|---|---|---|
| 1 | 마을 게시판 | 주문 시스템 |
| 2 | 우물 | 물뿌리개 자동 충전 |
| 3 | 반디네 부엌(식당) | 방치 판매 |
| 4 | 숲의 다리 | 숲 맵 개방(채집) |
| 5 | 빵집 화덕 | 가공품(밀가루·버터·치즈) 구매 |
| 6 | 온실 | 계절 무관 재배 6칸 |
| 7 | 정령의 등불탑 | 축복 슬롯 +1, 야간 마을 점등 |
| 8 | 축제 광장 | 엔딩 + 주간 축제(판매 이벤트) |

### 3.6 정령(반딧불) 시스템
- **저녁~밤 시간대**(실제 시각 19~06시 또는 게임 내 밤 토글)에 맵 곳곳에 반딧불 출현 → 탭해서 수집.
- 반딧불 = 보조 재화. **정령의 등불**에서 축복 구매: `성장 가속 2배(2시간)`, `판매가 +20%(2시간)`, `★확률 2배(다음 수확 10회)`, `즉시 성장(전설급, 비쌈)`.
- 수상한 죽·요리를 등불에 공양하면 반딧불 획득 (실패작 재활용처).

### 3.7 시간·계절·오프라인
- **계절**: 실제 7일 = 게임 한 계절 (28일 주기, 봄→여름→가을→겨울). 계절별 재배 작물·맵 색감·이벤트 변화.
- **오프라인 정산**: 재접속 시 `경과시간 = now - lastSave`를 작물 성장·동물 생산(상한 3개)·식당 판매(상한 12시간 분)에 적용 → **"돌아온 걸 환영해요!" 정산 팝업**(수확 대기 N개, 식당 수익 N골드).
- 시간 조작(기기 시계 되돌리기)은 `lastSave > now`면 무시로 단순 방어.

## 6. 기술 설계

### 스택
- **Vanilla JS (ES Modules) + Canvas 2D** — 빌드 도구 없음, `python3 -m http.server`로 즉시 실행, GitHub Pages 배포.
- PWA: `manifest.json` + Service Worker(에셋 프리캐시) → 홈 화면 설치, 오프라인 실행.
- 세이브: `localStorage` 단일 JSON + `lastSave` 타임스탬프. 30초 자동 저장 + visibilitychange 저장.

### 렌더링
- 논리 해상도 고정(뷰포트 세로 640×360급, 타일 32px), `imageSmoothingEnabled=false` + CSS `image-rendering: pixelated`, 정수 배율 스케일 → 픽셀 선명도 보장.
- **맵 = 대형 배경 일러스트 1장 + 오브젝트 스프라이트 레이어(y-sort) + 충돌 그리드(JSON 2D 배열)**. AI로 타일셋을 조각조각 만드는 대신 맵 전체를 한 장으로 생성 — 일관성 문제 원천 차단. 충돌 배열은 디버그 오버레이(격자 표시 + 탭 토글 에디터)로 코드 내에서 편집.
- 게임 루프: `requestAnimationFrame` + delta time. 성장 로직은 렌더와 분리된 타임스탬프 기반(오프라인 계산과 동일 코드 경로).

### 파일 구조
```
farm/
├─ index.html / manifest.json / sw.js
├─ css/style.css
├─ js/
│  ├─ main.js              # 부트스트랩, 씬 전환
│  ├─ engine/              # loop, camera, input(조이스틱), assets, save, audio
│  ├─ data/                # crops.js, animals.js, recipes.js, npcs.js, story.js,
│  │                       # buildings.js, maps/{farm,home,village,forest}.js(충돌+오브젝트 배치)
│  ├─ systems/             # farming, ranch, cooking, market, restaurant,
│  │                       # village, spirit, season, offline, quest, affinity
│  └─ ui/                  # hud, joystick, dialog, inventory, shop, recipeBook,
│                          # orderBoard, settlementPopup(오프라인 정산), cutscene
└─ assets/
   ├─ maps/  characters/  portraits/  crops/  animals/
   ├─ items/ dishes/ buildings/ objects/ ui/ cutscenes/
```

### 데이터 주도 설계
작물/동물/요리/건물/스토리는 전부 `data/*.js`의 선언적 객체로 정의 (예: `{id:'tomato', season:'summer', growMs:72e5, stages:2, regrow:true, seedPrice:40, sellPrice:88}`). 시스템 코드는 데이터를 순회만 함 → 밸런스 조정·콘텐츠 추가가 데이터 편집만으로 가능.

## 9. 검증 체크리스트

5. 에셋 파이프라인: 플레이스홀더 상태로 전체 플레이 가능 → 이미지 파일 추가 시 즉시 반영 확인
6. 밸런스: 첫 세션 10분 내 첫 요리 판매 도달, 1일차 500G 곡선 확인
